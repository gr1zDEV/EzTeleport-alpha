# Player-Facing Command Guide

Use this page as a ready-to-share command policy for your community.

## How teleport commands work

When a player uses a teleport command (like `/hub`):

1. A countdown may begin.
2. Movement or damage may cancel the teleport (server-dependent).
3. A cooldown may apply before reuse.

## Typical messages players may see

- Countdown updates with remaining time.
- Success message after teleport completes.
- Cancellation message if movement/damage interrupts.
- Cooldown message if command is reused too soon.

## Suggested player rules text

> Teleport commands can be cancelled by moving or taking damage. Stay still and safe until the countdown finishes.

## Staff FAQ responses

### “Why did my teleport cancel?”
It was likely interrupted by movement or damage during countdown.

### “Why can’t I use /spawn immediately again?”
A server cooldown is active for that command.

### “I ran /hub and nothing happened.”
Check for a chat/action bar message; if needed, contact staff to verify destination settings.
